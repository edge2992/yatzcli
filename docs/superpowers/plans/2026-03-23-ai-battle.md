# AI Battle Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** AI同士の対戦機能。Strategy パターンによる戦術プラグイン設計で、Greedy・Statistical・LLM 戦術を実装し、TUI 観戦モードと統計比較モードを提供する。

**Architecture:** `Strategy` インターフェースで戦術を抽象化。`RunBattle()` がゲームループを駆動し、`OnTurnDone` コールバックで TUI にストリーミング。LLM 戦術は `anthropic-sdk-go` で Claude API に直接接続。

**Tech Stack:** Go 1.22+, anthropic-sdk-go, bubbletea v2, cobra

**Spec:** `docs/superpowers/specs/2026-03-23-ai-battle-design.md`

---

## Phase 1: Strategy インターフェースと基本戦術（Greedy）

### Task 1: Strategy インターフェース定義

**Files:** `engine/strategy.go`, `engine/strategy_greedy.go`, `engine/ai.go`

- [x] **Step 1: `TurnAction` 構造体と `Strategy` インターフェースを定義**

`TurnAction` は `Type`（"hold" or "score"）、`Indices`（ホールド対象）、`Category`（スコア対象）を持つ。`Strategy` は `Name()` と `DecideAction()` を公開。

- [x] **Step 2: `GreedyStrategy` を実装**

常に即座にスコア。`bestCategoryForDice()` で最高得点カテゴリを選択。Hold は使わない。

- [x] **Step 3: `AIPlayer` を Strategy 対応にリファクタ**

`NewAIPlayerWithStrategy()` を追加。既存の `NewAIPlayer()` は `GreedyStrategy` をデフォルト使用。`PlayTurn()` を Strategy ベースのループに書き換え。

- [x] **Step 4: `AITurnResult` に `StrategyName` と `HoldHistory` を追加**

`HoldStep` 構造体（`Dice [5]int`, `Held []int`）を定義。各ホールドの履歴を記録。

## Phase 2: Statistical Strategy（期待値ベース）

### Task 2: 期待値計算と Statistical Strategy 実装

**Files:** `engine/expected_value.go`, `engine/strategy_statistical.go`

- [x] **Step 1: ホールド組み合わせ列挙**

5ビットマスクで全32パターンを事前計算（`allHoldCombinations`）。

- [x] **Step 2: `expectedValue()` を実装**

フリーダイスの全出目を列挙し、各パターンで `bestScoreForDice()` を取得して平均化。

- [x] **Step 3: `expectedValueWithBonus()` で上段ボーナス補正**

上段合計が閾値未満かつ残りカテゴリで到達可能な場合、`UpperBonusValue * 0.1` の補正を加算。

- [x] **Step 4: `StatisticalStrategy.DecideAction()` を実装**

3回目ロール後は即スコア。それ以外は全ホールド組み合わせの期待値を計算し、即スコアの得点を上回るものがあれば採用。

## Phase 3: LLM Strategy + Persona System

### Task 3: LLM Strategy 実装

**Files:** `bot/strategy_llm.go`, `bot/persona.go`

- [x] **Step 1: `Persona` 構造体と `LoadPersona()` パーサー**

Markdown ファイルから `Name`、`Personality`、`Strategy`、`Catchphrase` を抽出。日本語・英語の両セクション見出しに対応。

- [x] **Step 2: 同梱ペルソナの作成**

`personas/aggressive.md`（アタッカー）、`personas/defensive.md`（ディフェンダー）、`personas/gambler.md`（ギャンブラー）。

- [x] **Step 3: `LLMStrategy` を実装**

`anthropic-sdk-go` で Claude API に接続。システムプロンプトにペルソナ情報を注入。ユーザープロンプトにゲーム状態を提供。JSON レスポンスをパースして `TurnAction` を返却。

- [x] **Step 4: フォールバック処理**

API エラー・パースエラー・無効アクション時は `GreedyStrategy` にフォールバック。ホールドインデックスのバリデーション（0-4 範囲チェック）。

## Phase 5: TUI Spectator Mode

### Task 5: 観戦 TUI

**Files:** `cli/spectator.go`

- [x] **Step 1: `spectatorModel` と `RunSpectator()` を実装**

チャネル経由で `AITurnResult` を受信。`specResultMsg` / `specTickMsg` / `specDoneMsg` のメッセージ型。

`history` の `AITurnResult` を順に `Scorecard.Fill()` してリアルタイムスコアを表示。
